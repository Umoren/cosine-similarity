The Mental Model Framework

Core Principle

Cosine similarity measures what type of thing vectors represent, not how much of that thing they contain.

Direction = "What"

  • Concept or category
  • Pattern or relationship
  • Type or style
  • Semantic meaning
  • Qualitative properties

Magnitude = "How Much"

  • Intensity or strength
  • Frequency or count
  • Scale or size
  • Quantitative properties
  • Absolute values

The Decision Framework

Quick Assessment Questions

  1. What are you comparing? Concepts/patterns (direction) or amounts/intensities (magnitude)?
  2. Does scale matter? If yes, consider Euclidean distance. If no, use cosine similarity.
  3. Are you looking for conceptual similarity? Use cosine similarity.
  4. Do absolute values matter for your task? If yes, avoid cosine similarity.

Quick Reference Cheat Sheet

When to Use Cosine Similarity

Text Analysis: Topics, sentiment direction, writing style, semantic search
Recommendations: User preferences, content similarity, product matching
Search & Discovery: Semantic similarity, document matching, visual search
Classification: Pattern recognition, category similarity, intent detection
Embeddings: Word vectors, image features, user behavior vectors
Clustering: Conceptual grouping, topic modeling, customer segmentation
Social Analysis: User similarity, content affinity, network analysis
Creative AI: Style transfer, genre classification, artistic similarity

When NOT to Use Cosine Similarity

Financial Values: Stock prices, revenues, costs, budgets, salaries
Physical Measurements: Temperatures, distances, weights, speeds
Quality Scores: Performance ratings, quality metrics, test scores
Exact Quantities: Inventory levels, population counts, precise amounts
Critical Thresholds: Safety limits, medical values, compliance levels
Image Properties: Brightness, contrast, resolution, color intensity
Time-Sensitive Values: Real-time prices, urgent quantities, deadlines
Rate Comparisons: Conversion rates, success rates, error rates

Critical Decision Point

Always ask: "Does the magnitude represent meaningful information that shouldn't be normalized away?"

If yes → Use Euclidean distance or other magnitude-preserving metrics

If no → Use cosine similarity for directional/conceptual comparison

The Ultimate Decision Framework

Use Cosine Similarity When:

  • Comparing concepts, not quantities
  • Looking for patterns and relationships
  • Building recommendation systems
  • Performing semantic analysis
  • Magnitude differences are irrelevant
  • Focus is on "what type" not "how much"

Use Other Metrics When:

  • Exact values matter for decisions
  • Comparing quantities or measurements
  • Safety or compliance thresholds exist
  • Financial amounts are critical
  • Scale represents important information
  • Precision is more important than patterns
